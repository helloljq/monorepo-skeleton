## 使用说明

### 何时需要 ADR

- 选择关键技术栈 (数据库、框架、语言)
- 重大架构变更 (微服务拆分、缓存策略)
- 跨团队影响的技术决策
- 需要记录权衡取舍的选择

### 新增 ADR

1. 创建文件: `NNNN-<决策标题>.md` (编号从 0001 递增)
2. 使用模板:

```markdown
# NNNN. 决策标题

日期: YYYY-MM-DD

## 状态

已接受

## 背景

描述问题背景和约束条件。

## 决策

描述做出的决策。

## 后果

描述决策带来的影响（正面和负面）。
```

3. 更新本索引

### 状态说明

- **已接受**: 决策已通过并实施
- **已废弃**: 决策不再适用
- **已取代**: 被新决策取代（标注新编号）

### 编号规则

- 从 `0001` 开始递增
- 编号一旦分配不可复用
- 废弃的决策保留编号，状态标记为"已废弃"
